索引结构

索引结构类似PGM，多层的索引被分成若干块，每一块是一个完整的索引，查询可以从块的最顶层顶路由到块的最底层，全程保持在块内。

线程组织

每32个线程组成一个warp负责若干查询，一个线程块有32个warp，共1024个线程。一个块中至少有一个warp负责从全局内存读取查询。其余warp每个负责索引结构的一层。查询从索引的顶层到底层经过若干warp，形成一条流水线,流水线的最后一层将结果写入全局内存。理想情况下，流水线每个周期输入一组查询，输出一组结果，流水线中每层负责的查询向下一层路由一步。

数据结构

如今的显卡基本都有至少64KB的共享内存，此后的推导和计算以此为前提

索引块描述

  • 块内流水线数量
  • 流水线级数，块内索引层数
  • 负责读取全局内存的warp数
  • 查找区域宽度

级间缓存

级间缓存位于共享内存中，存储层与层间路由的中间结果。每轮开始线程从缓存中读取输入，进行计算，在每轮结束时写入下一级的缓存。每个级间缓存有 个位置，每个位置有2个4字节值，分别是和，级间缓存大小$intersz = \frac{32}{l} 8,8 \le intersz \le 128pipecnt levelcnt = 32 - readercnttotintersz = intersz pipecnt levelcnt, 128B=816 \le totintersz < 128 32 = 4KB$

一个warp中每个线程共享级间缓存中的一个位置，存在两种访问方式和对应的数据排列方式：

  • 个线程分2次读取两种值，两种值分别连续存储
  • 个线程中的两半一次性读取两种值 ，之后shuffle到另一半，两种值间隔存储

按网上的资料，shuffle的速度貌似不如读共享内存，但还要试验一下

模型和数据

模型和数据在开始时从主存加载到共享内存，减去级间缓存的占用，还有至少60KB的空间可用
